# 出席の自動受付とスコア自動集計

定例会・セミナー等の出席を受け付けて、**スコアに自動反映**する仕組みです。

```
QR/合言葉で出席 → 参加記録DB（誰が・いつ・何に出たか）→ 自動集計 → スコアDB
```

## 設計の要点：スコアに直接 +1 しない

出席のたびにスコアへ加算するのではなく、**常に参加記録から再計算**します。そのため：

- **二重送信しても増えません**（同じ人・同じイベントは1件にまとまる）
- 間違えたら**参加記録を1行消すだけでスコアも自動で戻ります**
- 「誰が何回出たか」の証跡が残るので、表彰時に根拠を示せます

## 使うDB

| DB | 役割 | ID |
|---|---|---|
| Branch イベントDB | 開催予定・合言葉・受付の開閉 | `3a776a17-0aae-814d-8066-e4c4161e9961` |
| Branch 参加記録DB | 1行＝会員×イベント（出席/欠席/申込） | `3a776a17-0aae-8123-89ea-dbd65a7295e7` |

> 置き場所は「運営・講師マニュアル」ページ配下です。**Notion上で自由に移動して構いません**
> （IDは変わらないので動作に影響しません）。

## 自動で埋まるスコア項目

| イベントの種別 | 自動で入るスコア項目 |
|---|---|
| 定例会 | 定例会参加数 |
| スキルアップ講座 | スキルアップ講座受講数 |
| リーディング会 | リーディング会参加数 |
| ロープレ | ロープレ参加数 |
| マルシェ・イベント | マルシェやイベント参加数 |
| zoom解放日 | zoom解放日の出席率（%）＝ 出席回数 ÷ その四半期の開催回数 |

> **⚠️ スプレッドシート連携との併用に注意**
> 上記の項目は「参加記録が正」です。スプレッドシート側からは**入力しないでください**
> （入力すると次回の再集計で上書きされます）。月収など他の項目は従来どおりスプシで入力可能です。

## 運営の手順

### 1. イベントを作る（開催前）
1. 「Branch イベントDB」で ＋新規
2. 入力：**イベント名／種別／開催日／合言葉**
3. **受付中** にチェック（これが出席受付のスイッチ）

合言葉は「その回だけの言葉」にしてください（例：`branch0724`）。受付を締めたら
**受付中のチェックを外します**。以降その合言葉では登録できません。

### 2. 参加者に案内する
出席ページのURLを共有します。

```
https://mayotesoh.github.io/Branch-site/checkin
合言葉つきリンク（QR向け）:
https://mayotesoh.github.io/Branch-site/checkin?code=branch0724
```

`?code=` を付けたURLをQRコードにすると、**読み取るだけで合言葉が自動入力**されます
（対面イベントは会場に掲示、オンラインはチャットに貼るだけ）。

### 3. 参加者の操作
合言葉とお名前を入れて送信するだけ。**会員名簿と自動照合**され、出席が記録されます。
氏名・占い師名のどちらでも照合でき、空白の有無は無視されます。

### 4. 欠席の記録（任意）
参加記録DBで該当行の**状態を「欠席」**にします（または手動で行を追加）。
出席率の分母や、今後の参加率分析に使えます。

## 連続参加ボーナス（皆勤賞）

GASの `awardStreakBadges` を実行すると、**直近3ヶ月すべての定例会に出席**した会員へ
会員DBの `称号` に **「皆勤賞」** を付与します。

- 月次の自動実行にしたい場合：Apps Script の「トリガー」で `awardStreakBadges` を月1回に設定
- 定例会が1回も開催されていない月がある場合は、誤付与を防ぐため判定をスキップします

## 手動での再集計

`recomputeAllScores` を実行すると、**今期の全会員**のスコアを参加記録から再計算します。
参加記録をまとめて手直しした後などに使ってください。

## セットアップ（GAS）

1. Apps Script に **`Attendance.gs` を新規追加**（[gas/Attendance.gs](../gas/Attendance.gs) を貼り付け）
2. **`Code.gs` を差し替え**（出席受付のルーティングを追加済み）
3. スクリプトプロパティに `NOTION_TOKEN` があることを確認
4. **イベントDB・参加記録DB・スコアDB・会員DB** をインテグレーションに「コネクト」
5. **デプロイ → デプロイを管理 → 編集 → 「新バージョン」→ デプロイ**

### 動作確認
1. イベントDBにテスト用イベント（合言葉つき・受付中ON）を作る
2. `/checkin` で合言葉＋自分の名前を送信
3. 参加記録DBに行ができ、スコアDBの該当項目と合計点が増えることを確認
4. 参加記録を削除 → 再集計でスコアが戻ることを確認

## QR・リンクの発行（主催者用ツール）

出席用のQRとリンクは、専用ツールで**合言葉を入れるだけ**で作れます。
別途QR作成サービスを使う必要はありません。

- ツールURL：`https://mayotesoh.github.io/Branch-site/checkin/qr`（ナビ非掲載・ブックマーク推奨）

### 手順
1. イベントDBで合言葉を決めて「受付中」にする
2. `/checkin/qr` を開き、合言葉（任意でイベント名）を入力 →「QR・リンクを表示」
3. QRコードと出席リンクが出る

### 対面イベント
「🔍 大きく表示」で画面いっぱいにQRを映す、または印刷して会場に掲示。
参加者がスマホのカメラで読み取り → 名前を入れて出席。

### オンライン（Zoom・LINEなど）
「コピー」ボタンでリンクをコピーし、**Zoomのチャットやグループに貼り付け**。
参加者がクリックして名前を入力すれば出席が記録されます（PCでもスマホでもOK）。

> どちらの場合も、参加者は「合言葉＋お名前」を送るだけ。名簿と自動照合され、
> スコア（定例会参加数など）に自動加点されます。二重送信しても増えません。
