# 会員スコア（ランキング）運用ガイド

コミュニティ会員のスコアを Notion で一元管理します。エクセル「ランク付け.xlsx」の
配点をそのまま移行し、合計点は**自動計算**されます。

## 構成

| DB | 役割 | ID |
|---|---|---|
| Branch 会員DB | 会員名簿。**表示のみ**：`今期得点`／`通算得点`（自動集計）・`称号`・`スコア公開` | `ca1b82cb-70c3-4995-b15b-362181c387cd` |
| Branch スコアDB | **1行 = 会員 × 四半期**。入力と計算はすべてこちら。`合計点` を自動計算 | `3a776a17-0aae-81a0-8708-d1e6c4d245f7` |

- **会員DBには入力項目も計算式も置きません**（追加したのは4つだけ）。メンバー管理の操作性を落とさない設計
- 置き場所：Notion の「運営・講師マニュアル」ページ配下（**自由に移動して構いません**）
- 会員ページを開くと、その人の `スコア履歴` がリレーションで並びます

### 会員DBの見た目を汚さないコツ
Notion は**ビューごとにプロパティを表示/非表示**にできます。普段のメンバー管理ビューでは
スコア系4つを非表示にし、「スコア確認」ビューだけで表示すれば、従来通りの操作感を保てます。

## 100人規模でも大丈夫か

| 項目 | 100人での規模 | 判定 |
|---|---|---|
| スコアDBの行数 | 100人×4期＝**年400行**（5年で約2,000行） | ✅ Notionは数千行でも快適 |
| 会員DBの行数 | 100行（スコアで増えない） | ✅ 問題なし |
| **1四半期の入力セル数** | 100人×18項目＝**1,800セル** | ⚠️ ここが実質のボトルネック |

行数ではなく**入力の手間**が効いてきます。対策：

- **四半期の行作成は自動化済み**（下記スクリプト。100人でも数十秒）
- **スプレッドシートから一括貼り付け**：Notionのテーブルビューは、表計算からコピーした
  セル範囲をそのまま貼り付けできます。大量入力はこれが最速
- **自動集計に寄せる**：`受講講座` リレーションからロールアップで受講数を自動カウントするなど、
  手入力そのものを減らす（拡張候補）

## 配点（エクセルから移行）

| 項目 | 倍率 |
|---|---|
| 月収(万円) | ×10 |
| 認定試験の点数 / 外部マルシェ出店経験 / ロープレ時お客様役回数 / コミュニティ紹介数 / 動員数 / 集客人数 | ×5 |
| スキルアップ講座受講数 / マルシェやイベント参加数 / リーディング会参加数 / ロープレ参加数 | ×3 |
| zoom解放日の出席率 / 定例会参加数 | ×1 |
| 即行動 / 我流じゃない / ポジティブ / 陰の気を出していない / レスポンス早い | 〇=5 / △=3 / ✖=0 |

> **⚠️ 月収の単位**：`月収(万円)` は**万円単位**で入力してください（例：月収12万円 → `12`）。
> 円単位で入れると倍率10で桁が飛びます。単位を変えたい場合は数式の調整が必要です（対応可）。

## 運用フロー

1. **四半期の開始時**：繰り越しスクリプトを実行（下記）。現在 `2026-Q3` は26名分**作成済み**
2. **月次**：同じ行の数値を更新して加点していく（※行は増やさない）
3. **四半期の終了時**：確定 → 表彰 → 次の期へ繰り越し
4. **リセット＆履歴**：期ごとに行が分かれるので、**リセットしつつ過去スコアは永久に参照可能**

### 四半期の繰り越し（自動）

```bash
cd Branch-site
node scripts/quarter-rollover.mjs 2026-Q4 --dry-run   # 確認のみ（書き込みなし）
node scripts/quarter-rollover.mjs 2026-Q4             # 実行
```

このスクリプトが自動で行うこと：
1. 旧四半期の行の `今期` チェックを外す
2. 活動中の会員全員ぶん、新四半期の行を作成（`今期` ON）
3. すでに行がある会員はスキップ（**何度実行しても安全**）

※ `退会`・`見込み` ステータスの会員は新規作成をスキップします（`SKIP_STATUS` で変更可）。

### 今期得点と通算得点の仕組み
- スコアDBの `今期` チェックが入っている行だけが `今期点` に計上されます
- 会員DBの `今期得点` ＝ `今期点` の合計、`通算得点` ＝ 全期間 `合計点` の合計
- 繰り越しスクリプトが `今期` を自動で切り替えるため、手作業は不要です

> **⚠️ 配点を変更するときの注意**
> Notion API の制約で数式から他の数式を参照できないため、`合計点` と `今期点` に
> **同じ配点式が二重に入っています**。倍率を変える場合は**両方の数式を更新**してください
> （片方だけだと今期得点だけズレます）。依頼いただければこちらで対応します。

## 推奨ビュー設定（Notion側で作成してください）

| ビュー名 | 設定 |
|---|---|
| **今期** | フィルタ：四半期 = 2026-Q3 ／ 普段はこれだけ見る |
| **今期ランキング** | 今期フィルタ ＋ 合計点で降順ソート |
| **会員別履歴** | 会員でグループ化（各自の推移を追う） |
| **過去** | フィルタ：四半期 ≠ 今期（アーカイブ用） |

## 公開方針（重要）

サイトは静的公開のため、**載せた情報は誰でも見られます**。スコアの内訳には月収が
含まれるため、公開は次の方針とします。

- ✅ 公開する：**順位・表示名・合計点のみ**
- ❌ 公開しない：**内訳（特に月収）**
- 会員DBの `スコア公開` にチェックが入っている人だけを掲載する運用を想定
- 本人向けの内訳確認は、公式LINEで自分のスコアを返す方式が安全（今後実装可）

## 今後の拡張候補

- **サイトのランキングページ**（`/ranking`）：順位・合計点を掲載。称号バッジも表示
- **自動加点**：講座受講・ロープレ参加をフォーム/LINEから記録 → GASでスコアDBに自動加算（人手削減）
- **受講数の自動カウント**：会員DBの `受講講座` リレーションからロールアップで自動集計（手入力を削減）
- **成長賞**：前期比の伸び率で表彰し、上位固定化を防ぐ
- **区分別ランキング**：運営陣／一般を分けて公平に
