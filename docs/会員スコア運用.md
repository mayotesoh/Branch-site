# 会員スコア（ランキング）運用ガイド

コミュニティ会員のスコアを Notion で一元管理します。エクセル「ランク付け.xlsx」の
配点をそのまま移行し、合計点は**自動計算**されます。

## 構成

| DB | 役割 | ID |
|---|---|---|
| Branch 会員DB | 会員名簿。`通算得点`（自動集計）・`称号`・`スコア公開` を追加済み | `ca1b82cb-70c3-4995-b15b-362181c387cd` |
| Branch スコアDB | **1行 = 会員 × 四半期**。各項目を入力すると `合計点` を自動計算 | `3a776a17-0aae-81a0-8708-d1e6c4d245f7` |

- 置き場所：Notion の「運営・講師マニュアル」ページ配下（**自由に移動して構いません**）
- 会員ページを開くと、その人の `スコア履歴` がリレーションで並びます

## 配点（エクセルから移行）

| 項目 | 倍率 |
|---|---|
| 月収(万円) | ×10 |
| 認定試験の点数 / 外部マルシェ出店経験 / ロープレ時お客様役回数 / コミュニティ紹介数 / 動員数 / 集客人数 | ×5 |
| スキルアップ講座受講数 / マルシェやイベント参加数 / リーディング会参加数 / ロープレ参加数 | ×3 |
| zoom解放日の出席率 / 定例会参加数 | ×1 |
| 即行動 / 我流じゃない / ポジティブ / 陰の気を出していない / レスポンス早い | 〇=5 / △=3 / ✖=0 |

> **⚠️ 月収の単位**：`月収(万円)` は**万円単位**で入力してください（例：月収12万円 → `12`）。
> 円単位で入れると倍率10で桁が飛びます。単位を変えたい場合は数式の調整が必要です（対応可）。

## 運用フロー

1. **四半期の開始時**：その期の行を会員分作成（現在 `2026-Q3` は26名分**作成済み**）
2. **月次**：同じ行の数値を更新して加点していく（※行は増やさない）
3. **四半期の終了時**：確定 → 表彰。次の期の行を新規作成
4. **リセット＆履歴**：期ごとに行が分かれるので、**リセットしつつ過去スコアは永久に参照可能**

### 行数が増えても大丈夫な理由
26名 × 4期 = 年104行。5年で約520行ですが、Notion は数千行でも快適です。
**「1行＝会員×四半期」に固定**し（加点イベントごとに行を作らない）、下のビューで絞れば
日常的に見るのは常に26行だけです。

## 推奨ビュー設定（Notion側で作成してください）

| ビュー名 | 設定 |
|---|---|
| **今期** | フィルタ：四半期 = 2026-Q3 ／ 普段はこれだけ見る |
| **今期ランキング** | 今期フィルタ ＋ 合計点で降順ソート |
| **会員別履歴** | 会員でグループ化（各自の推移を追う） |
| **過去** | フィルタ：四半期 ≠ 今期（アーカイブ用） |

## 公開方針（重要）

サイトは静的公開のため、**載せた情報は誰でも見られます**。スコアの内訳には月収が
含まれるため、公開は次の方針とします。

- ✅ 公開する：**順位・表示名・合計点のみ**
- ❌ 公開しない：**内訳（特に月収）**
- 会員DBの `スコア公開` にチェックが入っている人だけを掲載する運用を想定
- 本人向けの内訳確認は、公式LINEで自分のスコアを返す方式が安全（今後実装可）

## 今後の拡張候補

- **サイトのランキングページ**（`/ranking`）：順位・合計点を掲載。称号バッジも表示
- **自動加点**：講座受講・ロープレ参加をフォーム/LINEから記録 → GASでスコアDBに自動加算（人手削減）
- **受講数の自動カウント**：会員DBの `受講講座` リレーションからロールアップで自動集計（手入力を削減）
- **成長賞**：前期比の伸び率で表彰し、上位固定化を防ぐ
- **区分別ランキング**：運営陣／一般を分けて公平に
