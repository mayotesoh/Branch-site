# 講座決済（会員／非会員で価格を出し分け）

サイト内で講座の申込・決済まで完結し、**会員が購入すると受講履歴と得点に自動反映**されます。

```
申込ページ → 会員番号＋氏名で照合 → 価格決定 → 決済
   → 受講講座に追加 ＋ 参加記録DBに1行 → スコアに自動加点
```

## なぜ「会員番号＋氏名」なのか

合言葉（全員共通）では**誰が買ったか特定できない**ため、受講履歴や得点の自動反映ができません。
会員番号なら個人を特定でき、氏名も照合することで**なりすましを防止**します。

- 番号の表記ゆれ（`br001` / `BR-001`）や氏名のスペース有無は自動で吸収します
- 番号が合っていても氏名が違う場合は**拒否**されます

## 使うDB・プロパティ

| DB | 追加したもの |
|---|---|
| Branch 会員DB | `会員番号`（BR-001〜 で26名に採番済み）、`講座申込`（リレーション） |
| Branch 講座DB | `会員価格`、`非会員価格`、`決済対象`（チェックで販売開始） |
| Branch 講座申込DB（新設） | 申込・売上の記録。`決済ID` で二重登録を防止 |

> **重要**：講座を販売するには、講座DBで **`決済対象` にチェック** し、
> **`会員価格` と `非会員価格` を入力**してください。未設定の講座は申込ページに出ません。

## 運営の手順

1. 「Branch 講座DB」で対象の講座を開く
2. `会員価格` と `非会員価格` を入力（例：50000 / 80000）
3. `決済対象` にチェック → 次のサイト反映で申込ページに表示されます

会員番号は会員DBの `会員番号` 欄にあります。会員へ番号をご案内ください（自由に変更できます）。

## 申込者の流れ

1. `https://mayotesoh.github.io/Branch-site/apply` を開く
2. 講座を選び、**会員 / 一般** を選択
3. 会員は**会員番号とお名前**を入力（金額が会員価格に切り替わります）
4. 申し込む → 決済 → 完了ページ

## 購入後に自動で起きること（会員のみ）

- 会員DBの **`受講講座`** にその講座が追加される（受けた講座のリストアップ）
- **参加記録DB**に「種別＝スキルアップ講座／状態＝出席」で1行作成
- 既存の自動集計で **`スキルアップ講座受講数`（×3点）** が加算され、合計点に反映

> 得点はスコアへ直接足さず、参加記録を経由します。これにより二重加算が起きず、
> **返金時は参加記録を消せば得点も自動で戻ります**。

## Stripe キーが未設定のあいだ

`STRIPE_SECRET_KEY` が未登録でも申込ページは動きます。その場合は決済に進まず、
**「お申し込みを受け付けました。お支払い方法は担当より追ってご案内します」** と表示され、
講座申込DBに `未決済` で記録されます。キーを登録すればそのまま決済フローに切り替わります。

## セットアップ（GAS）

1. Apps Script（**Branch Webアプリ**）に **`CoursePayment.gs` を新規追加**
2. **`Code.gs` を差し替え**（`course_checkout` と `course_confirm` のルートを追加済み）
3. **Branch 講座申込DB** をインテグレーションに接続（「サイト・講座」ページごと接続でも可）
4. Stripeキーを使う場合はスクリプトプロパティに `STRIPE_SECRET_KEY` を登録
5. **デプロイ → デプロイを管理 → 編集 → 「新バージョン」→ デプロイ**
