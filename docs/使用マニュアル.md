# Branch 予約システム 使用マニュアル

最終更新: 2026-06-25

このマニュアルは「予約する人（利用者）」と「運営する人（管理者）」の両方向けです。

---

## A. 利用者向け（予約のしかた）

### A-1. Webサイトから予約する

1. 予約ページを開く → **https://mayotesoh.github.io/Branch-site/reserve**
   （サイト各所の「LINEで予約」ボタン、または上部メニュー「予約」からも）
2. **お名前・メールアドレス・予約内容・予約日・予約時間**を入力
3. 「**この内容で予約する**」を押す
4. 「予約を受け付けました」と表示されれば完了

### A-2. LINEから予約する（対話式）

1. 公式LINEを友だち追加 → **https://lin.ee/SvCKDYoj**
2. **リッチメニュー（予約する）をタップ**、または「**予約**」と送信
3. 表示されたボタンから**コンテンツを選択**
4. **希望日時を入力**（例: `2026-07-01 14:00`）
5. 「予約を受け付けました」と返信が来れば完了

## B. 管理者向け（運営のしかた）

### B-1. 予約の確認

予約はすべて Google スプレッドシートに記録されます。
列の並び：

```
タイムスタンプ | userName | userId | メールアドレス | date | time | コンテンツ
```

- 新しい予約ほど下の行に追加されます
- LINE予約はメールアドレスが空欄（LINE名・userIdが入ります）

### B-2. コンテンツ種別を変更したいとき

「体験講座」などの選択肢を増減・変更する場合、**2か所**を合わせて直します。

1. **サイト側**: `src/consts.ts` の `CONTENT_TYPES` を編集
2. **LINE側**: `LineBot.gs` の `LINE_CONTENT_TYPES` を同じ内容に編集

その後、
- サイト側 → Git に push（自動で再デプロイ）
- LINE側 → Apps Script で保存し**再デプロイ**

### B-3. 予約時間の選択肢を変えたいとき（Webフォーム）

`src/pages/reserve.astro` の冒頭、時間枠を生成しているループを編集：

```js
for (let h = 10; h <= 19; h++) {        // 開始〜終了の「時」
  for (const m of ['00', '30']) {        // 何分刻みか
    timeSlots.push(`${String(h).padStart(2, '0')}:${m}`);
  }
}
```

編集後、push すれば反映されます。

### B-4. GASコードを更新したときの手順（重要）

コードや設定値を変えても、**再デプロイしないと反映されません**。

1. Apps Script でコードを編集 → **Ctrl + S で保存**
2. **デプロイ → デプロイを管理 → 鉛筆(編集) → バージョン「新バージョン」→ デプロイ**
   - ※URLは変わりません

### B-5. シートの見出しをリセットしたいとき

`Code.gs` の **`resetHeaders()`** を実行すると、見出しを最新仕様に作り直せます。
**⚠️ シートの中身が全部消えます。** 残したいデータがある場合は実行しないでください。

## C. トラブルシューティング

### C-1. LINEで「次の配信までお待ちください」等の定型文が返る
→ 自動応答がONになっています。
**LINE公式アカウントマネージャー → 設定 → 応答設定** で：
- 応答モード: **Bot**
- Webhook: **オン**
- 応答メッセージ: **オフ**

### C-2. LINEで「予約」しても無反応
原因の多くは**トークン未設定／再デプロイ忘れ**。
1. `LineBot.gs` に `testToken()` を貼って実行 → ログが
   - `200` … トークンOK → **再デプロイ**を確認（B-4）
   - `401` … トークンが無効 → 正しい「チャネルアクセストークン（長期）」を再設定
   ```javascript
   function testToken() {
     const res = UrlFetchApp.fetch('https://api.line.me/v2/bot/info', {
       headers: { Authorization: 'Bearer ' + LINE_CHANNEL_ACCESS_TOKEN },
       muteHttpExceptions: true
     });
     Logger.log(res.getResponseCode() + ': ' + res.getContentText());
   }
   ```
2. それでもダメなら、LINE Developers →「Messaging API設定」→ **Webhook URL** が
   GASの `/exec` URL と一致しているか、「**検証**」で `Success` か確認。

### C-3. Webフォームが「送信に失敗しました」
- GASウェブアプリのアクセス設定が「**全員**」になっているか
- `src/consts.ts` の `GAS_URL` が最新のデプロイURLか
- ブラウザのキャッシュ（**Ctrl + F5** で再読込）

### C-4. 列がずれて記録される
→ シートの見出しが旧仕様の可能性。B-5 の `resetHeaders()`、または手動で
列を `タイムスタンプ / userName / userId / メールアドレス / date / time / コンテンツ` に揃える。
