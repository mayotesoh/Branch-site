# Branch 予約システム 仕様書

最終更新: 2026-06-25

## 1. 概要

Branch の予約システムは、**2つの入口**から予約を受け付け、いずれも同じ
Google スプレッドシートに記録します。

| 入口 | 利用者 | 取得項目 |
| --- | --- | --- |
| ① Webサイト予約フォーム | サイト訪問者 / LINE内ブラウザ(LIFF) | 名前・メール・コンテンツ・日付・時間 |
| ② LINE 対話式予約 | LINE友だち | （LINE名）・コンテンツ・日付・時間 |

## 2. システム構成図

```
[利用者]
   │
   ├─(A) サイト /reserve フォーム ──┐
   │     fetch(text/plain JSON)     │
   │                                ▼
   └─(B) LINE トーク ──▶ [LINE Messaging API] ──▶ [GAS Web App doPost] ──▶ [Google スプレッドシート]
          リッチメニュー/「予約」      Webhook            (ルーター)
```

- **フロント**: Astro 製の静的サイト（GitHub Pages）
- **バックエンド**: Google Apps Script（Web App / `/exec`）
- **データ保存**: Google スプレッドシート
- **対話チャネル**: LINE 公式アカウント（Messaging API）

## 3. データ仕様（スプレッドシート）

最初のシートに、1予約＝1行で追記。

| 列 | 項目 | 例 | 備考 |
| --- | --- | --- | --- |
| A | タイムスタンプ | 2026/06/25 23:10:00 | サーバー受信時刻（自動） |
| B | userName | 春名 渼月 | 名前 / LINE表示名 |
| C | userId | Ufdb… / web-1719… | LINEユーザーID。Web予約は一時ID |
| D | メールアドレス | example@mail.com | Web予約のみ。LINE予約は空欄 |
| E | date | 2026-07-01 | YYYY-MM-DD |
| F | time | 14:00 | HH:MM（LINEで日付のみ入力時は空欄） |
| G | コンテンツ | 体験講座 | 予約内容 |

## 4. コンテンツ種別

サイト・LINE共通（変更時は §7 参照）。

```
体験講座 / 養成講座 / 個別セッション / 練習会・ロープレ / 鑑定 / キャリア相談 / その他
```

## 5. GAS API 仕様

エンドポイント: GAS ウェブアプリURL（`https://script.google.com/macros/s/XXXX/exec`）
- アクセス: 全員 / 実行ユーザー: 自分
- `doPost` が受信内容で処理を振り分ける（ルーター）

### 5.1 Web/LIFF 予約（POST）

- リクエスト: `Content-Type: text/plain;charset=utf-8`、ボディは JSON 文字列
  ```json
  { "userName": "...", "userId": "...", "email": "...", "content": "...", "date": "YYYY-MM-DD", "time": "HH:MM" }
  ```
- 必須: `userId` / `content` / `date` / `time`
- レスポンス:
  | 状態 | 内容 |
  | --- | --- |
  | 成功 | `{"status":"success"}` |
  | 失敗 | `{"status":"error","message":"…"}` |

> **CORS対策**: `application/json` ではなく `text/plain` で送ることで
> プリフライト（OPTIONS）を回避する（GAS は任意ヘッダーを返せないため）。

### 5.2 LINE Webhook（POST）

- LINE Messaging API から `{ "events": [ ... ] }` が届く
- `doPost` は `events` 配列を検知して対話処理（§6）へ
- レスポンス: `{"status":"ok"}`（LINEには200応答すればよい）

### 5.3 動作確認（GET）

- `/exec` をブラウザで開くと `{"status":"ok","message":"Branch 予約API は稼働中です。"}`

## 6. LINE 対話フロー（状態遷移）

会話状態は GAS の `ScriptProperties` に `state_<userId>` で保存。

```
(開始) リッチメニュータップ / 「予約」送信
   │  → step = awaitContent
   ▼
コンテンツ選択（クイックリプライ or 自由入力）
   │  → step = awaitDate, content 保存
   ▼
希望日時を入力（例: 2026-07-01 14:00）
   │  → 日時をパースし、シートへ追記
   ▼
完了メッセージ返信、状態クリア
```

- 日時パースは `YYYY-MM-DD HH:MM` / `YYYY/MM/DD` / 日付のみ等に対応
- 読み取れない場合は再入力を促す

## 7. ファイル構成

### サイト（リポジトリ）
| パス | 役割 |
| --- | --- |
| `src/pages/reserve.astro` | 予約フォーム＋LIFF初期化＋送信処理 |
| `src/consts.ts` | `GAS_URL` / `LIFF_ID` / `CONTENT_TYPES` / `LINE_URL` |
| `public/richmenu.svg` | リッチメニュー画像テンプレート（2500×843） |

### GAS（Apps Script プロジェクト）
| ファイル | 役割 |
| --- | --- |
| `Code.gs` | doPostルーター・フォーム処理・シート追記・`resetHeaders()`・doGet |
| `LineBot.gs` | LINE対話処理・状態管理・LINE API呼び出し・`LINE_CHANNEL_ACCESS_TOKEN` |
| `RichMenu.gs` | リッチメニュー作成（API利用時。任意） |

> リポジトリ `gas/` 配下は**参照用コピー**。実体は Apps Script 側。
> 変更したら Apps Script に反映し、**再デプロイ**が必要。

## 8. セキュリティ / 制約

- **チャネルアクセストークン**は GAS 内のみに保持（サイト/公開リポジトリに置かない）
- LINE Webhook の**署名検証（X-Line-Signature）は未実装**（簡易版）。
  本格運用時は `LineBot.gs` に署名検証を追加推奨
- GAS ウェブアプリは「全員」アクセス。受信データのバリデーションは実装済み
- 予約の**重複/空き枠チェックは未実装**（記録のみ）。必要なら拡張
