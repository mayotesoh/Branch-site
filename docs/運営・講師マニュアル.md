# Branch 運営・講師マニュアル

最終更新: 2026-07-07

このマニュアルは **運営メンバー・講師** が、Notion を使ってサイトの内容
（ブログ・講座・会員・予約）を管理するための手引きです。
**むずかしい操作はありません。作業はすべて Notion の中だけ**で完結します。

---

## 0. 全体像（まずこれだけ）

- サイトの内容は **Notion に書くと、自動でサイトに反映**されます。
- 反映は **1日2回（朝9時・夜21時ごろ）自動**。すぐ出したいときは担当者が「今すぐ反映」できます（§6）。
- みなさんが触るのは **Notion だけ**。GitHub や設定ファイルは触りません（§1-3）。

```
あなた → Notionに書く → （自動ビルド）→ サイトに反映
```

対象データベース（Notion）:

| DB | 使う人 | 用途 |
| --- | --- | --- |
| ブログ記事DB | 講師・運営 | 記事を書く |
| 講師DB | 運営 | 講師プロフィール |
| 講座DB | 運営 | 講座の追加・編集 |
| 会員DB | 運営 | 会員管理・対応ログ |
| 予約管理DB | 運営 | 予約の確認・ステータス |

---

## 1. メンバーへの渡し方（アクセス権限）

### 1-1. Notion に招待する
1. 対象の Notion ページ／データベースを開く
2. 右上の **「共有」** をクリック
3. 相手の **メールアドレス**を入力して招待
4. 権限を選ぶ（下記の目安）

> まとめて渡したい場合は、各DBが入っている **親ページ（またはワークスペース）** に招待すると、
> 中のDBすべてにアクセスできます。

### 1-2. 権限の目安

| 相手 | 渡すDB | 権限 |
| --- | --- | --- |
| **講師** | ブログ記事DB（＋講師DBは閲覧） | 編集可 |
| **運営** | 会員DB・予約管理DB・講座DB・ブログ記事DB・講師DB | 編集可 |
| 見せるだけ | 任意 | 閲覧のみ / コメント可 |

- 「編集可」＝ 書き込み・変更ができる
- 「コメント可」＝ 中身は見られるがコメントだけ
- 「閲覧のみ」＝ 見るだけ

### 1-3. 触らないもの（重要）
次のものは**運営・講師は触りません**（サイトが壊れる原因になります）:
- GitHub のコード / `.github` / 設定ファイル
- Google Apps Script（GAS）
- **Notion のトークン（`ntn_...`）** … これは機密情報。誰にも共有しない

## 2. 【講師向け】ブログの書き方

**占い師それぞれが、自分のアカウントで自分の記事を書けます。**

### 2-1. 記事を書く手順
1. Notion で **「ブログ記事DB」** を開く
2. **「＋ 新規」** で新しいページ（記事）を作成
3. 次の項目を入力：
   | 項目 | 内容 |
   | --- | --- |
   | **タイトル** | 記事のタイトル |
   | **著者** | 「講師DB」から自分を選ぶ（← 著者別ページに反映されます）|
   | **公開日** | 表示・並び順に使われます |
   | **抜粋** | 一覧に出る短い説明（1〜2行）|
   | **タグ** | ジャンル（例：タロット・運勢）|
   | **本文** | ページの中に自由に執筆（見出し・箇条書き・画像OK）|
4. 書き終えたら、**「公開状態」を「公開」** にする
5. これで完了。次の自動反映（またはすぐ反映）でサイトに出ます

### 2-2. 下書き・非公開にしたいとき
- **公開状態を「下書き」**（＝「公開」以外）にしておけば、サイトには出ません。
- 完成したら「公開」に変えるだけ。

### 2-3. 画像について（注意）
- 本文やカバーに画像を入れるときは、**画像のURL（外部リンク）** を使うのが安全です。
- Notion に直接アップロードした画像は、時間が経つと表示URLが切れて**サイトで見えなくなる**ことがあります。
  （恒久対応が必要なら運営・開発担当に相談）

### 2-4. よくある質問
- **Q. URLを自分で決めたい** → 「slug」欄に半角英数字を入れる（空なら自動）。
- **Q. 反映されない** → 「公開状態＝公開」になっているか確認。反映は1日2回自動（§6）。
- **Q. 他の人の記事も見える** → 同じDBを共有しているため見えます（編集はしないルールで運用）。

## 3. 【運営向け】会員管理

### 3-1. 会員DBの考え方
- **1行＝1会員**。増えても縦に増えるだけで、横に広がりません。
- 基本情報（氏名・占い師名・区分・入会月・占術・ステータス）は**一覧の列**に。
- 詳しい経過は、その会員の**ページを開いた中（本文）**に「対応ログ」として書きます。

### 3-2. 対応ログの書き方（統一ルール）
会員ページを開き、本文に **日付＋内容** で追記します：
```
6/22：プランニングセッション受講。月収10万円が目標。イベント出演希望。
```
- 新しい記録は下に追加していく
- 誰が書いても同じ形式になるよう、**「日付：内容」** で統一

### 3-3. ステータスの運用
「会員ステータス」を状況に応じて変更：
- **見込み** → **活動中** → **要フォロー** → **退会検討** → **退会**

### 3-4. 増えても見やすくする「ビュー」（最初に1回だけ設定）
会員DBで **「＋ ビューを追加」** から作成しておくと便利です：
1. **ボード**：グループ化を「会員ステータス」→ カンバンで進捗管理
2. **要フォロー**：フィルターで「会員ステータス＝要フォロー」だけ表示
3. **区分別**：グループ化を「区分」（会員／運営／講師）

> ビューは「見せ方」を変えるだけ。**データは1つ**なので二重管理になりません。

---

## 4. 【運営向け】予約の確認

- 予約は **「予約管理DB」** に自動で入ります（サイトのフォーム・LINE予約の両方）。
- 各予約の **「ステータス」** を運用：**未対応 → 確定 → 完了 / キャンセル**
- 会員と紐づけたい場合は、予約の「予約履歴（会員）」リレーションで会員を選択。

---

## 5. 【運営向け】講座の追加・編集

1. **「講座DB」** を開く
2. 既存講座を編集、または **「＋ 新規」** で追加
3. 入力項目：
   | 項目 | 内容 |
   | --- | --- |
   | 講座名 / 種別 | 種別＝「養成講座」or「セッション」|
   | カテゴリ | 養成講座／キャリア支援／体験講座／練習会／収益化／メンタル |
   | 担当講師名・担当講師 | 講師名（＋講師DBへのリレーション）|
   | 説明・期間・時間・提供方法 | 各項目 |
   | カリキュラム | ページ本文に箇条書き（養成講座のみ）|
   | 表示順・公開 | 並び順と、公開ON |
4. **「公開」にチェック**すると、次の反映でサイトに出ます。

> 新しいカテゴリを追加したときだけ、アイコン等の調整に開発対応が必要です。

---

## 6. サイトへの反映（公開）の仕組み

- **自動**：1日2回（朝9時・夜21時ごろ）に最新のNotion内容へ更新。
- **今すぐ反映**（担当者のみ）：
  1. GitHub → 対象リポジトリ → **Actions** タブ
  2. 「Deploy to GitHub Pages」→ **Run workflow**
  - ※これは GitHub 操作なので、**触れる担当者を1名決めておく**のがおすすめです。

> 「書いたのにすぐ出ない」は正常です。次の自動反映を待つか、担当者に反映を依頼してください。

---

## 7. 困ったとき

| 症状 | 対処 |
| --- | --- |
| 記事・講座が出ない | 「公開状態＝公開」/「公開ON」か確認。反映は1日2回（§6）|
| 画像が消えた | Notion直アップの画像は期限切れの可能性 → 外部URL画像に（§2-3）|
| Notionが編集できない | 権限が「閲覧のみ」かも → 招待者に「編集可」を依頼（§1）|
| 予約がNotionに入らない | 開発・運営担当に連絡（GAS/連携の確認が必要）|
