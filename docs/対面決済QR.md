# 対面決済（現場でQRを表示して決済）

マルシェ・イベント等で、その場で金額を入力してお客様のスマホで決済してもらう仕組みです。
**カードリーダーや端末は不要**。スタッフのスマホ／タブレット1台で完結します。

- スタッフ用URL：`https://mayotesoh.github.io/Branch-site/pay`
- ※このページはナビに載せていません。スタッフだけがURLで開きます（ブックマーク推奨）

## 使い方（現場）

1. スタッフが `/pay` を開く
2. **品目**（例：タロット鑑定30分）と**金額**を入力 →「QRコードを表示」
3. 画面にQRが出る → **お客様が自分のスマホのカメラで読み取る**
4. お客様のスマホで決済（カード／Apple Pay／Google Pay）
5. 完了すると**スタッフ画面が自動で「✓ お支払いが完了しました」に変わる**
6. 「別の会計をする」で次のお客様へ

お客様側には「お支払いありがとうございました」の画面が出るので、
スタッフはそれを見せてもらえば対面でも確認できます。

## 記録

すべて「**Branch 対面決済DB**」（`3a776a17-0aae-810a-842f-dbde06f5058c`）に残ります。

| プロパティ | 内容 |
|---|---|
| 品目・金額・担当・メモ | 入力内容 |
| 決済状態 | 未決済／支払済み（入金で自動更新） |
| 決済ID・日時 | 照合用 |

売上の集計はこのDBを見ればOKです。

## 安全のための制限

- 金額は **100円〜300,000円** の範囲のみ（誤入力による高額決済を防止）
- QRは **30分で失効**します
- 通信が一時的に切れても、数秒ごとに自動で再確認します

## セットアップ（GAS）

1. Apps Script（**Branch Webアプリ**）に **`OfflinePayment.gs` を新規追加**
2. **`Code.gs` を差し替え**（`offline_checkout` / `offline_status` を追加済み）
3. **Branch 対面決済DB** をインテグレーションに接続
4. スクリプトプロパティに **`STRIPE_SECRET_KEY`** を登録（対面決済は決済必須）
5. **デプロイ → 新バージョン → デプロイ**

> Stripeキーが未設定の間は、QR表示時に「決済がまだ有効化されていません」と出ます。
> キーが届いたら登録するだけで使えるようになります。
