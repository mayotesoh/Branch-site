# Branch × Notion 連携設計

最終更新: 2026-07-05

方針（確定）:
- **講座・講師・ブログ = Notion を正データ（管理元）** に一本化
- **予約 = 現行の GAS + スプレッドシートを維持**し、GAS から Notion「予約管理DB」にも同期
- 着手順: **① ブログ基盤（実装済み・ローカルMD）→ ② Notion差し替え → ③ 講座Notion化 → ④ 予約Notion同期**

---

## 1. 全体像

```
Notion ワークスペース（共同運営）
 ├ 講師DB     ─┐  リレーション
 ├ 講座DB     ─┼───────────────┐
 ├ ブログ記事DB ┘               │  ビルド時にAPI取得
 └ 予約管理DB ←── GAS が予約時に追記   │
                                     ▼
                        Astro build → GitHub Pages（静的サイト）
                             ▲ 定期(cron) ＋ 手動(workflow_dispatch)
```

静的サイトのため、Notionの変更は **再ビルドで反映**（後述 §5）。

## 3. 共同運営（占い師ごとの記事）

- ブログ記事DBを**1つ共有**し、各占い師を Notion に**ゲスト招待（編集権限）**
- 記事の「著者」は講師DBへのリレーションで指定 → サイトが著者別ページ `/blog/author/<id>` を自動生成
- 各自が自分のアカウントで記事を書き、「公開状態=公開」にすると次のビルドで反映

⚠️ **注意**: 同一DBの共同編集では、権限上はお互いの記事も編集可能（Notion仕様）。
厳密な分離が必要な場合は運用ルールで対応（本設計は信頼ベースの共同運営を前提）。

---

## 4. サイト側の実装方針（差し替え）

現状のブログはローカルMarkdown（`src/content/blog/*.md`）で稼働中。
Notion化は **コンテンツコレクションの loader を差し替えるだけ**で、
一覧・記事・著者別ページ（`src/pages/blog/**`）はそのまま使える設計。

```
現在:  glob loader（ローカルMD）
  ↓ 差し替え
Notion: notion-astro-loader 等で Notion記事DB を取得
        （schema は title/author/publishDate/excerpt/cover/tags/draft を踏襲）
```

必要なもの（ユーザー準備）:
1. Notion インテグレーション作成 → **APIトークン**（`ntn_...`）
2. 各DBをそのインテグレーションに**接続（コネクト）**
3. 各DBの **Database ID**

上記を受け取り次第、`.env`（`NOTION_TOKEN` / `NOTION_BLOG_DB` 等）に設定し、
GitHub Actions の Secrets にも登録してビルドで参照する。

> トークンは**公開リポジトリに置かない**。`.env` は gitignore、CI は Secrets 経由。

---

## 5. 反映（再ビルド）の仕組み

静的サイトのため、Notion更新は再ビルドで反映される。2系統を用意:

- **定期ビルド**: GitHub Actions の `schedule`（例: 1日2回）
- **手動公開**: `workflow_dispatch`（「今すぐ反映」ボタン相当）

将来的に即時反映が必要なら、Notion Webhook / ボタン→Actions API 呼び出しで自動化可能。

---

## 6. 予約のNotion同期（GAS拡張）

現行フロー（フォーム / LINE対話 → GAS → スプシ）は**変更しない**。
`appendReservation` の後に **Notion API へも POST** する処理を追加するだけ。

```
予約発生
  → スプシに追記（現状のまま）
  → Notion 予約管理DB に1ページ作成（追加）
```

- Notionトークンは **GAS の Script Properties** に保存（コードに直書きしない）
- 失敗してもスプシ記録は成立するよう、Notion同期は try/catch で握りつぶす（記録優先）

## 7. 実装状況・進め方

1. **[完了]** ブログ基盤（一覧 / 記事 / 著者別ページ）
2. **[完了]** Notion 4DB 作成・インテグレーション接続
3. **[完了]** ブログを Notion 取得に差し替え（`src/lib/notion.ts`）
   - `@notionhq/client@2.2.15` + `notion-to-md` + `marked`
   - トークンは `.env`（ローカル）/ GitHub Secret `NOTION_TOKEN`（CI）
   - 定期再ビルド（1日2回）＋手動（Actions の Run workflow）で反映
4. **[要検討]** 講座ページを Notion 講座DB 連動に置き換え
   - 現 `courses.astro` はカリキュラム/カテゴリ/提供方法など**リッチ構造**。
     最小スキーマのままだと情報が欠落するため、講座DBの**プロパティ拡張が必要**（§8）。
5. **[コード実装済 / 要デプロイ]** GAS に予約→Notion同期を追加
   - `gas/NotionSync.gs`：`syncReservationToNotion()` を追加
   - `gas/Code.gs`：`appendReservation()` 末尾で try/catch 呼び出し（記録優先）
   - **デプロイ手順**（Apps Script側で実施）:
     1. `NotionSync.gs` を新規追加、`Code.gs` を更新
     2. プロジェクト設定 → スクリプトプロパティに `NOTION_TOKEN` を登録
     3. 予約管理DB をインテグレーションに「コネクト」
     4. `testNotionSync()` を実行して成功を確認 → **再デプロイ**

## 8. 講座DB 拡張案（移行前の検討事項）

現ページの情報を Notion で保持するための追加プロパティ案:

| プロパティ | 型 | 用途 |
| --- | --- | --- |
| カテゴリ | セレクト | 養成講座 / キャリア支援 / 体験講座 / 練習会 / 収益化 / メンタル |
| 対象・ねらい | テキスト | target/desc |
| 期間・時間 | テキスト | 8時間×2回 / 90分 等 |
| 提供方法 | テキスト | オンライン可 / セミナー 等 |
| アフターフォロー | チェックボックス | 2か月無料フォロー有無 |
| 備考 | テキスト | 受講条件など |
| カリキュラム | ページ本文 | 箇条書きで記載 |

> この拡張を行えば、現ページと同等の見た目を Notion 管理で再現可能。
> 拡張は API で非破壊的に追加できるため、承認後すぐ着手できる。

### ブログ記事の書き方（運営者向け）
1. Notion「ブログ記事DB」で新規ページを作成
2. **タイトル / 著者（講師DBから選択）/ 公開日 / 抜粋 / タグ** を入力し、本文を書く
3. **公開状態を「公開」** にする
4. 反映は自動（1日2回）。すぐ反映したい場合は
   GitHub → Actions → 「Deploy to GitHub Pages」→ **Run workflow**

> slug は任意（空ならNotionのページIDが使われる）。URLを固定したいときだけ入力。
> カバー・本文中の画像は Notion アップロードだと表示URLが期限切れになるため、
> **外部URL画像**の利用を推奨（恒久運用は画像ダウンロード対応を別途検討）。
